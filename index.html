What is RTChat?

RTChat is a serverless peer-to-peer communication library that enables real-time chat, audio, and video calls without requiring your own backend infrastructure. It uses WebRTC for direct peer-to-peer connections and MQTT only for initial signaling (discovering other users), making it incredibly lightweight and efficient.

Once peers discover each other via MQTT, all communication happens directly between browsers using WebRTC. This means no server costs, no data storage, and end-to-end encrypted connections by default.

Quick Start

Get started with RTChat in just a few lines of code:

import { MQTTRTCClient } from '@rtchat/core'; const client = new MQTTRTCClient({ name: 'MyName', topic: { room: 'myroom' } }); // Listen for peer connections client.on('connectedtopeer', (user) => { console.log('Connected to', user); }); // Listen for chat messages client.on('chat', (message, sender) => { console.log(`${sender}: ${message}`); }); // Send a message to all connected peers client.sendRTCChat('Hello everyone!');

Video & Audio Calls

RTChat includes built-in support for peer-to-peer video and audio calls. Make calls directly between browsers without any server in the middle:

// Make a video call const { start, end } = client.callUser('PeerName', { video: true, audio: true }); start.then(({ localStream, remoteStream }) => { // Display streams in video elements localVideo.srcObject = localStream; remoteVideo.srcObject = remoteStream; }); // Handle incoming calls client.on('call', (peerName, callInfo, promises) => { if (confirm(`Accept call from ${peerName}?`)) { return promises.start.then(({ localStream, remoteStream }) => { localVideo.srcObject = localStream; remoteVideo.srcObject = remoteStream; }); } });

How It Works

RTChat uses a two-phase connection process:

  1. Discovery Phase: Peers connect to a public MQTT broker and announce their presence on a shared topic (room).
  2. Direct Connection: Once peers discover each other, they establish a direct WebRTC connection. All subsequent communication happens peer-to-peer.

This architecture means MQTT is only used briefly for signaling (~3kB per connection), and all actual data (messages, video, audio) flows directly between peers with no server in between.
